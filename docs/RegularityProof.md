# 定理 R' 証明下書き — 評価時定数性推論の可容性

作成: 2026-07-16。`docs/RegularityNotes.md` の候補定理 R' の紙上証明の
第一稿。機械検証済みの前提(η-在庫等)は LibStrictness.lean が典拠。

## 0. 設定

ABCFHL 型の cubical 型理論 T₀ を固定する:
- 区間 I = 区間変数上の自由 De Morgan 代数
- transp の型付け: Γ, i:I ⊢ A type、Γ ⊢ u : A(i0) ⟹
  Γ ⊢ transpⁱ A u : A(i1)
- 各型形成子での transp 計算規則(Π/Σ/Path/Glue/基底型/HIT)
- **構文的定数性規則 S**(標準・健全): i ∉ FV(A) ⟹ transpⁱ A u ≡ u
- η 規則: Π・Σ・Path(**本カーネルの conv が全て検証済み** —
  strictFunEta / strictSigEta / strictPathEta)

**規則 (R)[判断的定数性]**: Γ, i ⊢ A ≡ A₀ かつ i ∉ FV(A₀) ならば
transpⁱ A u ≡ u。

**規則 (R')[正規形側条件・アルゴリズム版]**: i ∉ FV(nf A) のとき
簡約 transpⁱ A u ⟶ u。

## 1. 補題 1(定式化の同値)

周囲簡約の正規化と合流を仮定すると、
「∃A₀. A ≡ A₀ ∧ i ∉ FV(A₀)」 ⟺ 「i ∉ FV(nf A)」。

*証明*. (⇐) A ≡ nf A を A₀ に取る。(⇒) 正規形の一意性より
nf A = nf A₀。簡約は自由変数を導入しないので
FV(nf A₀) ⊆ FV(A₀) ∌ i。∎

## 2. 補題 2(代入安定性)— 証明の心臓

σ : Δ → Γ、Γ, i ⊢ A ≡ A₀、i ∉ FV(A₀) ならば
Δ, i ⊢ Aσ ≡ A₀σ かつ i ∉ FV(A₀σ)。

*証明*. 等式判断は代入で閉じる。i は transp が束縛する次元であり
σ の定義域にも値にも現れ得ない(値は Δ 上の項で、i は α-変換で
新鮮に取れる)。よって
FV(A₀σ) ⊆ (FV(A₀) \ dom σ) ∪ ⋃ₓ FV(σx) は i を含まない。∎

*系*. (R)-redex の集合は代入で閉じ、(R)-簡約は代入と可換。
(規則の不安定性 — Agda が φ を定義時固定にする理由として想定される
懸念 — は生じない。)

## 3. 補題 3(型保存)

前提下で A(i0) ≡ A₀ ≡ A(i1)。よって u : A(i0) ⟹ u : A(i1) であり
(R) は型を保つ。∎

## 4. 補題 4(局所整合性 — 臨界対の収束)

A が判断的に i-定数で、かつ型形成子 F の構造的 transp 節も適用可能な
とき、両契約は T₀+(R) で可換:

| F | 構造的契約 | 収束 |
|---|---|---|
| 基底型 (ℕ,ℤ,⊤,⊥,S¹,T²,U) | u | 即座に一致 |
| Π | λx. transp B (u (transp A⁻ x)) | 副線も i-定数 → (R) で λx. u x → **η(Π)** で u |
| Σ | (transp (fst u), transp (snd u)) | (R) で (fst u, snd u) → **η(Σ)** で u |
| PathP | ⟨j⟩ transp-系 | (R) + **η(Path)** で u |
| HIT(点・セル別) | 構成子ごとに副線へ | 副線 i-定数 → (R) で構成子引数が不動 → 文字通り一致(η 不要) |
| Glue | transpGlue | 下記 4.1 |

η-在庫は本カーネルで**全て機械検証済み**。
注意: この表は「T₀+(R) は Σ-η を**導出**する」ことも示す
((R) と Σ-構造節から (fst u, snd u) ≡ u)— 
η を持たない基礎の上では (R) は η を強制する。

### 4.1 Glue 節(残る非自明ケース)

判断的 i-定数な Glue 線への transpGlue の出力が u と可換であること。
- 実装上はショートカットが先行するため**アルゴリズムは決定的**で
  この対は実体化しない。
- 等式理論としては: 定数 Glue 線ではブランチ線・基底線とも i-定数、
  transpGlue の各副輸送が (R)-可約、ファイバー補正は退化データ上で
  可縮性の中心に落ちる — の 3 段で u に収束する見込み。
  **精密化は残作業(本稿の主要未完部)**。
- 回避策(採用方針): 拡張を**決定的 NbE 戦略**(定数性検査を最優先)
  として提示し、判断的等式をその誘導 PER と定義する。このとき
  必要なのは「T₀ の各等式が妥当」(健全性)のみで、
  同じ計算を形成子ごとに一度行えば足りる。

## 5. 主定理(可容性)と証明戦略

**定理**: T₀ + (R) は canonicity を持ち(閉自然数項は数字に評価)、
変換は NbE で決定可能。従って一貫。系: `J d refl ≡ d` が定義的。

*証明戦略*: 標準的な NbE 論理関係証明の拡張。
- 計算可能性モデル(構文的論理関係)では値=正規形であり、
  (R) は構成により妥当(本実装の quote∘eval がその構成的体現;
  300 定義 + 全ガードの一貫稼働が経験的証拠)。
- 代入安定性(補題 2)が基本補題の代入ケースを通す。
- 決定的戦略の採用(4.1)により合流は不要。

## 8. アーキテクチャ改訂(2026-07-17): 第2次外部レビューを受けた再設計

第2次レビューの中心指摘(3.1)は正しい: **¬Const(A) は置換に安定しない**
(それどころか「置換で定数化する」ことこそ本研究の動機)。したがって
「構造規則を非定数族に制限した T^≺」は置換に閉じた等式理論として
成立しない。さらに本日の機械実験(LibSwitchover.lean)で、修復
(切替時整合性の証明)が**不可能**であることが判明した:

### 8.1 switchover 整合性の機械的判定(新知見)

定数族では (R) と構造規則の両方が適用可能。両 contractum の収束を
前者ごとに機械検査した結果:

| 前者 | 構造 contractum(定数族に特殊化) | 収束? | probe |
|---|---|---|---|
| Π | `λx. transp B (u (transp A x))` | **是**((R)+fun-η) | `swPiD` |
| Σ(非依存/依存) | `(transp A (fst u), transp B' (snd u))` | **是**((R)+Σ-η) | `swSigD`, `swSigDepD` |
| HIT 構成子(list) | 構成子ごとの transp | **是**((R) のみ) | `swListD` |
| **PathP** | `⟨j⟩ comp A [j=0↦a, j=1↦b] (p@j)` | **否** | 負 `#guard`(LibSwitchover §4) |
| Glue | transpGlue 展開 | 否(同機構) | Path の失敗が残滓機構を目撃 |

**PathP の失敗が決定的**: 構造規則は端点補正の comp を残し、定数族では
その残滓が「充填次元について定数なチューブの hcomp」になる。カーネルに
hcomp-regularity はない(`trans refl refl ≢ refl` の負例と同じ機構、
かつ Swan/Sattler によりモデル論的にも要求できない)ので、残滓は
stuck。すなわち:

> **無制限の T₀+(R) は hcomp-regularity 型の等式
> `⟨j⟩ hcomp[j=0↦a, j=1↦b](p@j) ≡ p` を導出してしまう。**
> この等式はアルゴリズムが拒否し、標準モデルでの妥当性も疑わしい。
> ゆえに優先化(= (R) 先行の決定的評価戦略)は実装の便宜ではなく
> **理論的必然**であり、それは Glue だけでなく Path 型で既に生じる。

これはレビューが疑った「Lemma 4.4 の Path 行」が実際に誤りだった
ことを意味する(旧稿の critical-pair 表は Path を η で収束としていた
— comp 残滓を見落としていた)。レビューの精査に感謝すべき点。

### 8.2 正しい定義: 判断的等式 = アルゴリズム的等式

規則生成型の等式理論(制限つき)が置換に閉じない以上、拡張理論の
判断的等式は **NbE アルゴリズムの値の等式として定義する**(実装型
理論の標準的立場; Agda の definitional equality と同じ地位):

- 評価器 eval(環境 ρ 付き)、constancy 検査 const?、変換 conv の
  相互再帰(決定的)。transp ノードでは const? を先に試す(優先)。
- **置換安定性 = NbE 置換補題**: `eval(tσ, ρ) = eval(t, eval(σ, ρ))`。
  構造的帰納で証明され、**transp の場合も含めて成立する**: constancy
  検査は「族の値」(capp fam generic)に対する occurs-check であり、
  値のレベルでは tσ を ρ で評価しても t を σ∘ρ で評価しても同一の値
  に到達するから、検査結果も同一。レビューが要求した
  「eval(tσ) と eval(t)σ の coherence(構造規則→(R) 切替の場合を
  含む)」は、まさにこの補題の transp 節として証明される —
  切替は値レベルでは「起きない」(検査は常に置換適用後の値で行われる)。
- 旧 Definition(T^≺ = 規則制限)は**破棄**。「T^≺」の名は以後
  「優先化評価戦略が誘導するアルゴリズム的等式」を指す。
- レビュー 3.2(理論の Const と実装の Const の不一致)も同時に解消:
  もはや二つの定義はなく、検査はアルゴリズムの一部として一度だけ
  定義される。

   (基底正規化の仮定パッケージ + (R) 発火は項サイズ減少)、

## 9. D3: 不可能性定理の定式化と Glue 残滓の明示計算(2026-07-17)

### 9.1 定理(切替不整合、アルゴリズム水準 — 無条件)

**定理 N(機械検証済み)**: 本カーネルのアルゴリズム的等式において、
生成的文脈 (A : U, a b : A, p : Path A a b) で
```
⟨j⟩ hcomp^i A [j=0 ↦ a, j=1 ↦ b] (p @ j)   ≢   p        (†)
```
(LibSwitchover §4 の負 guard; 対照 probe により (†) 左辺の
well-typedness も機械確認済み)。

**系**: T₀ の構造 Path 規則(transp の Path 前者での展開)と (R) と
合同・推移律をすべて含む等式理論は (†) の左右を同定する。ゆえに
- そのような理論に対して本アルゴリズムは**完全になり得ない**;
- 「(R) を含み、構造規則を無制限に保ち、置換に閉じる」規則生成型
  理論は、hcomp-regularity 型の等式 (†) を必然的に含む。

導出: 定数族 Path A a b への transp に (R) を適用すると p、構造規則を
適用すると ⟨j⟩comp[...](p@j)(transp 部は (R) で消え hcomp 残滓のみ)。
推移律で (†) の両辺が等しくなる。∎

### 9.2 モデル論的裏付け(条件付き・引用ベース)

(†) の右辺=左辺を**判断的等式として**要求することは「すべての型で
端点定数な開箱の filler が退化する」ことを要求する。Sattler の指摘
(CCHM 宇宙の合成の非正則性)および Swan の分離定理は、標準
cubical 集合モデルの合成構造がこの性質を持たないことを示す。
したがって (†) を等式として含む理論は標準モデルでは解釈できない
見込みが高い(正確には「標準的な合成構造の選択では解釈できない」;
regularity を持つ合成構造の非存在そのものは型に依存する)。
論文では条件付き命題として述べ、Swan/Sattler を引用する。

### 9.3 Glue 残滓の明示計算(transpGlue、定数族、live 枝 1 本)

族 G := Glue [φ ↦ (T, w)] B(φ, T, w, B すべて i-自由)、u₀ : G。
transpGlue(kernel `Semantics.lean` の実装に即して):

1. δ := ∀i.φ = φ(定数なので live)。
2. t₁' := transp^i T u₀|_T — T 定数なので **(R) で u₀|_T に崩壊**。
3. fill_i := transp^j T(i∧j) u₀|_T — 同じく崩壊、チューブは
   i ↦ w.1(u₀|_T)、**i について定数**。
4. a₁' := comp^i B [δ ↦ w.1(u₀|_T)] (unglue u₀)
   — B 定数で transp 部は消え、**定数チューブ hcomp が残留**
   (境界整合: δ 上で unglue u₀ = w.1(u₀|_T) ✓、δ 外で stuck)。
   これは §9.1 の (†) と同一機構。
5. さらに fiber-point 補正(fiber Σ-型上の可縮性連結 hcomp)が
   live 枝ごとに追加の hcomp 残滓を積む。
6. 結果 glue [φ ↦ 補正点] (補正後の base) は u₀ と非 convertible。

結論: Glue の switchover 残滓は Path と同じ「定数チューブ hcomp」+
fiber 補正であり、Path の失敗 (†) が既に本質を目撃している。旧稿の
「Glue だけが臨界対の未解決点」という描像は誤りで、正しくは
「**hcomp を生成するすべての構造規則(Path, Glue)で切替不整合が
生じ、優先化が強制される**」。transp のみを生成する構造規則
(Π, Σ, HIT 構成子, 基底型)では収束する(機械検証済み、§8.1)。

### 10.1 NbE 置換補題(完全証明の骨格)

**補題 S**: 全ての項 t、代入 σ、環境 ρ について
`eval(tσ, ρ) = eval(t, eval(σ, ρ))`。

項 t の構造帰納。決定的評価器なので等式は値の同一性。
- 変数・λ・app・pair・…: 標準(閉包の環境合成)。
- **transp^i A u(核心節)**: 両辺とも族の値
  `F := capp (eval(A…)) (generic ℓ)` を計算する。左辺の族値は
  eval(Aσ, ρ)、右辺は eval(A, eval(σ,ρ)) に generic を適用したもの —
  帰納法の仮定(A は真部分項)により同一の値。constancy 検査
  usesLvl(ℓ, F) は**値のみの関数**なので判定が一致。
  - 検査成功: 両辺 = eval(uσ, ρ) = eval(u, eval(σ,ρ))(帰納法)。
  - 検査失敗: 構造節へ dispatch — head は同一値なので同一分岐、
    以降は部分評価の帰納法。∎(切替は値レベルで発生し得ない)
- hcomp/glue 節: 同様(系の面式評価も値関数)。

### 10.2 computability 述語(値上、型値帰納)

`Comp_T(v)` を型値 T の帰納で定義(Huber の述語の値版):
- Comp_nat(v) ⟺ v は数字項(の値)または v ≡ 数字項。
- Comp_Π, Comp_Σ, Comp_PathP: 標準(引数適用・射影・端点条件)。
- Comp_Glue: unglue 像の computability + 面上の枝 computability。
- Comp_U: 小さい型値のコード + その composition 構造の computability。
- HIT: 構成子閉包 + hcomp 閉包。

**再利用(A4 由来、無修正)**: 述語節そのもの、hcomp の computability
(合成構造の節は (R) に触れない)、単射性による well-definedness。

### 10.3 fundamental lemma(環境版)

**定理 F**: Γ ⊢ t : T、ρ が Γ-computable な環境なら
eval(t, ρ) は Comp_{eval(T,ρ)}。

typing 導出の帰納。基底規則の節は (A4) の証明と同一(環境版に
書き換えるだけ — 置換版でなく環境版なので §3.1 型の問題は不発生)。
**新規節は transp のみ**:
- 優先分岐(検査成功): 族値が ℓ-自由 ⟹ 両端の型値が同一の値 ⟹
  Comp_{T(0)} = Comp_{T(1)} は文字通り同じ述語。IH で
  eval(u,ρ) ∈ Comp ⟹ 結果(= その値)∈ Comp。型理論的等式への
  言及すら不要(値の同一性)— v2 の「等式閉包」議論より強く単純。
- 構造分岐(検査失敗): 基底証明の transp 節と同文。ただし内部の
  再帰的 vtransp 呼び出しが優先分岐に入る場合があるが、それは
  同じ定理 F の(値サイズに関する)帰納で処理される — 評価器の
  停止性(10.4)が帰納の well-foundedness を与える。

**系(canonicity)**: 閉じた t : nat の値は数字項。
**系(一貫性)**: アルゴリズム的等式で 0 ≢ 1(異なる値)。
**系(J-refl)**: J d refl の評価は優先分岐で d の値に到達(機械
検証: strictJReflD)。

### 10.4 評価器の停止性

well-typed 項上: (A1) + 「(R) 発火は評価図式のノードを厳密に減らす」
(u は transp ノードの真部分項; const? は A の評価 — 真部分項 — と
値の走査のみ)。優先分岐の追加は基底の呼び出し関係に新しい閉路を
作らない(§8.2 補題)。よって基底停止測度がそのまま通る。
注意: これは (A1) を「拡張評価器の再帰構造が基底と同型」という
観察で持ち上げる議論であり、論文では評価器の呼び出しグラフの
明示(付録図)を添える。

### 10.5 conv の健全性と「sound conversion checker」定理

conv(v, v') 受理 ⟹ v, v' は η/DNF 正規化で同一の正規形 ⟹
quote 像が T₀+(R) で導出可能な等式(各ステップが規則インスタンス)。
完全性は主張しない(§9.1 の系により、T₀+(R) 全体に対しては
**原理的に不可能**; 意味のある完全性主張は「アルゴリズム的等式は
それ自身に対し完全」という自明形か、将来の適切な部分理論に対して)。
